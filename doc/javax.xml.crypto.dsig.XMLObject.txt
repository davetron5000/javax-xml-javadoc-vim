*javax.xml.crypto.dsig.XMLObject_Description*

A representation of the XML Object element as defined in the W3C Recommendation for XML-Signature Syntax and Processing. An 
XMLObject may contain any data and may include optional MIME type, ID, and encoding attributes. The XML Schema Definition is 
defined as: 



<element name="Object" type="ds:ObjectType"/> <complexType name="ObjectType" mixed="true"> <sequence minOccurs="0" 
maxOccurs="unbounded"> <any namespace="##any" processContents="lax"/> </sequence> <attribute name="Id" type="ID" 
use="optional"/> <attribute name="MimeType" type="string" use="optional"/> <attribute name="Encoding" type="anyURI" 
use="optional"/> </complexType> 

A XMLObject instance may be created by invoking the newXMLObject(|javax.xml.crypto.dsig.XMLSignatureFactory|) method of the 
(|javax.xml.crypto.dsig.XMLSignatureFactory|) class; for example: 



XMLSignatureFactory fac = XMLSignatureFactory.getInstance("DOM"); List content = 
Collections.singletonList(fac.newManifest(references))); XMLObject object = factory.newXMLObject(content, "object-1", null, 
null); 

Note that this class is named XMLObject rather than Object to avoid naming clashes with the existing 
java.lang.Object(|java.lang.Object|) class.
